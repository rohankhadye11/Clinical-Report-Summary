# ClinicAI: Serverless Medical Extraction & Patient Portal

This project is an event-driven, serverless AI pipeline that automatically ingests unstructured medical documents, extracts key clinical data using specialized NLP, and visualizes it on a secure, patient-friendly Next.js web dashboard. It leverages Microsoft Azure's serverless and AI services for a highly scalable, secure, and cost-effective solution.

## 1. Project Overview

ClinicAI is a secure web application designed to bridge the gap between complex medical reports and patient comprehension. It comprises a modern frontend built with Next.js and a backend powered by Azure Functions, interacting with Azure Blob Storage for data persistence and Azure AI Text Analytics for health to extract and summarize medical jargon into readable insights.

## 2. Features

- **Secure Patient Access:** Patients can securely log in using their unique Record ID to access their specific medical data.
- **AI-Generated Summaries:** Displays a plain-English "Doctor's Note" generated by an extractive AI model.
- **Color-Coded Medical Entities:** Automatically categorizes and displays Symptoms (bullet points), Diagnoses (green badges), and Medications (yellow badges) to reduce cognitive load.
- **Automated Data Processing:** Uploading a medical document automatically triggers the backend pipeline to read, analyze, and structure the data without manual intervention.
- **Fault-Tolerant System:** Includes built-in quarantine queues for corrupted files to prevent system crashes.

## 4. Architecture

The application follows an event-driven, serverless architecture pattern, strictly separating the patient-facing frontend from the highly secure, automated backend AI logic.

![Architecture](https://github.com/rohankhadye11/Clinical-Report-Summary/blob/main/Architecture%20Snapshot.png) 

**Key Components & Interactions:**

- **Frontend:** Built with Next.js and Tailwind CSS, providing a responsive and accessible UI. It securely fetches processed records using server-side API routes so API keys are never exposed to the browser.
- **Azure Blob Storage:** Acts as the data lake. Unstructured files are dropped into `clinical-inbox`, and final structured CSVs are saved into `clinical-outbox`.
- **Azure Functions:** The backend engine. An event-driven Python function wakes up instantly when a new file lands in the inbox, routing it to the appropriate AI services.
- **Azure AI Services:**
  - *Document Intelligence* performs OCR on PDFs and images.
  - *Text Analytics for health* extracts medical entities, maps relationships, and generates an extractive summary.
- **Data Structuring:** The complex JSON from the AI is flattened into relational CSV tables (`summaries` and `entities`) making it perfectly formatted for the frontend to consume.

## 6. Usage

1. **Start the Application:** Run the Next.js development server locally (`npm run dev`) or access the deployed URL.
2. **Access Records:** On the main screen, enter a valid Record ID (e.g., `patient_006.txt`) into the search bar and click "View My Records".
3. **Review Summary:** Read the plain-English summary generated by the AI in the blue informational box at the top.
4. **Examine Specifics:** Review the categorized lists below to see specific symptoms noted, diagnoses made, and medications prescribed.
5. **Return to Search:** Click "Back to Search" to clear the dashboard and look up a different patient record.

## 7. Cost Considerations

This is a serverless architecture, meaning you pay only for actual compute time and API transactions.

- **Azure Functions:** Costs are based on execution time and total number of executions. The function only runs when a new document is uploaded.
- **Azure Blob Storage:** Costs are based on the volume of data stored (inbox/outbox files) and the number of read/write operations performed by the frontend and backend.
- **Azure Text Analytics for health:** Billed per text record processed by the NLP model.
- **Azure Document Intelligence:** Billed per page analyzed for OCR extraction.

Monitor your **Azure Cost Management** dashboard regularly to track usage and configure budget alerts.
